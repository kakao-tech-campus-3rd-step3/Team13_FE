# [KAN-100] 라우팅 가드 시뮬레이션(Protected/Verified Route)

## 상태 조합 & 기대 동작

| 상태                               | 경로     | 기대 동작                                  |
| ---------------------------------- | -------- | ------------------------------------------ |
| 미인증(user=null)                  | `/my`    | `/login`으로 리다이렉트 (`state.from=/my`) |
| 로그인+미검증(emailVerified=false) | `/my`    | `/email-cert`로 리다이렉트                 |
| 로그인+검증                        | `/my`    | 정상 렌더                                  |
| 세션 만료(sessionExpired=true)     | `/my`    | `/login?expired=1` + 토스트 표기           |
| 로그인 상태                        | `/login` | `from` 또는 `/my`로 이동 (루프 없음)       |

## 구성

- `PublicRoute` : 로그인 시 접근 금지(/login 등)
- `ProtectedRoute` : 인증 필요(+세션 만료 체크)
- `VerifiedRoute` : 이메일 검증 필요
- 페이지: `/login`, `/email-cert`, `/my`
- 스켈레톤: RouteSkeleton / 토스트: 로그인 만료 안내
- 인증 토큰 상태는 기존 `useAuthStore`가 계속 관리하며 API 인터셉터에서 활용

## 테스트

- MemoryRouter로 리다이렉트/렌더 검증
- 루프 발생 없음
